# 《数据库系统》课程大作业报告

## 一、需求分析

基于“速驰汽车销售公司”场景，系统覆盖售前（意向）、销售（订单）、库存（车辆状态）、售后（服务工单）及管理决策（统计报表）五大域。

## 二、E-R与逻辑设计

已抽取 10 个核心实体并建立完整主外键关系；关系模式满足 3NF，避免数据冗余和更新异常。

## 三、物理实现

完成数据库 `car_sales` 的建模、初始化数据、视图、索引、触发器、存储过程及复杂查询脚本。

## 四、实现难点与解决方案

1. **订单与库存一致性**：通过触发器+事务实现强一致。
2. **统计查询复用**：通过视图沉淀高频分析能力。
3. **应用安全性**：Python 端使用参数化查询，关键写操作显式事务回滚。

## 五、应用程序实现

Python 控制台应用包含：
- 销售前台：创建意向客户、创建销售订单、查询我的订单
- 库存管理：车辆入库、多条件库存查询、库存预警
- 报表中心：销售业绩榜、畅销车型排行、月度统计

已给出测试文档与关键用例，覆盖规则A/B/C与查询需求Q1-Q8。

## 七、个人总结

本项目完整演练了数据库课程设计闭环：从需求到模型、从脚本到应用、从功能到统计，强化了事务一致性、索引优化与SQL工程化能力。
